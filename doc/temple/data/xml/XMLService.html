InheritanceXMLService CoreEventDispatcher flash.events.EventDispatcher
Implements IDebuggable
Subclasses URLManager, XMLManager

Base class for loading XML data, with or without parameters. Extend this class to create a proper service for use in a MVCS-patterned application. The base class provides functionality for transferring an Object with parameters to the request, loading the XML, a virtual function for parsing the result, parsing a list, and error handling on response & parsing. When extending this class, the function protected function processData (inData:XML, inName:String) : void; has to be overridden & implemented to handle a successful load. When parsing a list of XMl nodes into an array of objects of one class, the parseList() function can be used.

destruct()method 
override public function destruct():void

Destroys the objects, all intern listeners will be removed.

When overriding this method, always call super.destruct() at the end!

If you want the object to be available for garbage collection make sure you:
  • Remove all event listeners on this object (use removeAllEventListeners on Temple objects).
  • Remove all event listeners from this object.
  • Set all non-primitive variables to null.
  • Set all references to this object to null in other objects.
When a Temple object is destructed a DestructEvent.DESTRUCT is dispatched from the object (if the object implements IDestructableEventDispatcher).

Note: Bear in mind that it is possible an object can be destructed more than once.
