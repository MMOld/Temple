Registry — class, package temple.debug
This class holds objects with their unique id to identify them when needed All objects added are stored in a weak-referenced-Dictionary with an autoincrementing id as value; When 'Temple.REGISTER_OBECTS' is set to true, all objects added to the Registry are also added to the Memory, to track their memory usage At the moment, the Registry is used for storing objects in the Memory class, using their unique id for Logging (tracing) and in the DebugManager
